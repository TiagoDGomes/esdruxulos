PHP

PHP é uma linguagem de programação interpretada, desenvolvida para uso principalmente em sites World Wide Web

Caracteristicas do PHP

  • Velocidade e robustez
  • Estruturado e orientação a objeto
  • Portabilidade - independência de plataforma - escreva uma vez, rode em qualquer lugar;
  • Tipagem fraca
  • Sintaxe similar a Linguagem C/C++ e o PERL

Em junho de 2004 foi lançada a versão 5 do PHP, introduzindo um novo modelo de orientação a objeto, incluindo a reformulação dos construtores e adição de destrutores (ver exemplo contextualizado em anexo), visibilidade de acesso, abstração de objeto e interfaces de objetos.

Outras características

Uma característica do PHP é em relação a arrays. Podemos ter arrays do tipo $a[0]="banana" , $a[1]="alface", como também arrays como $a["fruta"]="banana"; $a["verdura"]="alface". Assim, podemos definir chaves de arrays com nome que quiser. Além disso, são arrays dinâmicos, ou seja, são arrays que não tem tamanho fixo, que varia conforme o programador precise.

Outra característica do PHP é o suporte nativo para acesso a vários tipos de banco de dados, como MySQL, mSQL, Firebird, entre outros. O PHP possui funções e classes para o suporte a estes banco de dados, e sem muitas complicações para utilizá-las.

Nosso sistema

O nosso sistema utiliza a característica do array com chaves nomeadas para armazenar os dados que vêm do banco de dados. Como o resultado das consultas dos bancos retorna uma array com colunas e seus nomes, isso é bastante util, pois todos os valores dos atributos básicos das classes são armazenados um único array, e são as funções getters e setters que tem o trabalho de transformar em objetos os atributos que estão armazenados em variáveis simples, como IDs.

Uma das grandes dificuldades para implantar o suporte ao banco de dados ao site foi a precariedade em frameworks que se adequa a orientação a objetos no PHP. O projeto Lumine é um dos poucos que tem essa funcionalidade. Um projeto brasileiro que segue a linha de racionício do Hibernate do Java. Porém, como a documentação do Lumine estava incompleta, necessitávamos criar a nossa própria classe que integrava ao banco de dados. Basicamente, tivemos que fazer como se faz com o JDBC em Java, ou seja, na "unha".
